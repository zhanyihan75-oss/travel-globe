# AI Workflow

## Before Every Task

Before making any changes:

1. Read every Markdown document under /docs.
2. Understand the project vision.
3. Understand the PRD.
4. Understand the development guide.
5. Understand previous iterations.

Do not start coding until you fully understand the project.

---

## Development Mindset

Always think as:

- Product Designer
- UX Designer
- Front-end Engineer

Instead of only a programmer.

Every implementation should improve the product experience.

---

## Development Rules

Always:

✓ Small incremental changes

✓ Keep project runnable

✓ Preserve existing architecture

✓ Reuse components

✓ Explain implementation plan first

Never:

✗ Rewrite the project

✗ Introduce unnecessary libraries

✗ Add backend

✗ Add database

✗ Over engineer

---

## Response Format

Every response should contain:

1. Requirement Understanding

2. Solution

3. Files to modify

4. Complete Code

5. Verification

6. Suggested Next Step

---

## Decision Principle

If there are multiple implementation options,
always choose the one that is

- easier to understand
- easier to maintain
- easier to extend
- better for AI collaboration

instead of the most technically advanced solution.

---

## Product Principle

Every feature should answer:

"Does this make the Destination Page better?"

If not,

reconsider the implementation.
