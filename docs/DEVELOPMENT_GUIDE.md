# DEVELOPMENT_GUIDE.md

## Purpose

This document defines how Codex should develop this project.

## Before Every Task

1.  Read all files in `/docs`.
2.  Understand the current code before making changes.
3.  Propose the smallest possible implementation.

## Tech Stack

-   React
-   Vite
-   TypeScript
-   Three.js
-   GitHub Pages

## Rules

-   Keep implementations simple.
-   Prefer local JSON and local image folders.
-   No backend.
-   No database.
-   No unnecessary libraries.
-   No large refactors.
-   One feature at a time.
-   Keep components modular.
-   Preserve existing coding style.

## Output Format

For every task provide: 1. Goal 2. Files to modify 3. Why 4. Complete
code 5. Next recommended step

## Product Rule

Every feature should strengthen the Destination Page experience.
