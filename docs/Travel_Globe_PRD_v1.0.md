# Travel Globe PRD v1.0

## 产品定位

Travel Globe 是一个围绕 **Destination（目的地）**
构建的旅行产品。所有体验都从 3D 地球开始，最终汇聚到 Destination Page。

> 核心理念：Every Destination Tells Three Stories.
>
> -   My Story（我的旅行）
> -   Our Stories（未来预留：社区）
> -   Future Story（未来预留：AI 灵感）

------------------------------------------------------------------------

# 产品原则

1.  **Destination First**：所有功能围绕目的地页面。
2.  **Map First**：地图不是导航，而是产品本身。
3.  **Less is More**：保持极简信息架构。
4.  **MVP First**：优先完成可用版本，再逐步扩展。

# Codex 开发原则（必须遵循）

## 总体原则

Codex 在后续所有开发过程中必须遵循以下原则，不允许为了炫技引入复杂架构。

### 1. 优先简单实现

优先选择最简单、最稳定、最容易维护的方案。

### 2. MVP 优先

一次只完成一个小功能，不一次性生成完整系统。

### 3. 不过度设计

不要提前实现未来功能。

### 4. 本地数据优先

V1 不使用数据库，尽量使用本地文件、JSON 与图片目录。

### 5. 保持模块化

每个功能拆分为独立组件，避免一个巨大文件。

### 6. 保持现有代码风格

修改时尽量局部调整，不大规模重构。

### 7. 每次修改前

先阅读项目已有代码，再提出最小修改方案。

### 8. 技术栈固定

-   React
-   Vite
-   TypeScript
-   Three.js
-   GitHub Pages

不要主动引入 Next.js、Node 后端、数据库或复杂状态管理。

### 9. 产品一致性

任何新增功能必须回答：

> 这个功能是否服务于 Destination Page？

如果不是，应重新思考。

### 10. 输出规范

每次输出： - 修改目标 - 修改文件 - 修改原因 - 完整代码 - 下一步建议

避免一次修改多个无关功能。
